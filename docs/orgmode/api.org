Rust API
========

.. raw:: html

   <div class="cn-page-intro">
     <p>The <code>canonic</code> library crate is documented <strong>inside this Shibuya site</strong>
     via <code>sphinxcontrib-rust</code> (same pattern as rgpot): module pages are generated
     from the Rust sources during <code>./docs/build.sh</code> and appear under the toctree below.</p>
   </div>

Module map
----------

.. raw:: html

   <ul class="cn-mod-grid">
     <li><code>config</code><p><code>canonic.toml</code> / CLI prefix and optional <code>[jira]</code> settings.</p></li>
     <li><code>corpus</code><p>Walk and load <code>{prefix}-*.md</code> responses with front matter.</p></li>
     <li><code>check</code><p>Quality gate: prefix, sop, ids, team-generic closings.</p></li>
     <li><code>convert</code><p>Pandoc markdown ↔ Jira wiki markup.</p></li>
     <li><code>index</code><p>Tantivy reindex, BM25 search, near-duplicate pairs.</p></li>
     <li><code>lint</code><p>Vale CLI and in-process Harper grammar checks.</p></li>
     <li><code>doctor</code><p>Tooling presence (pandoc, Vale, Harper).</p></li>
     <li><code>jira_import</code><p>Free REST probe, import drafts, explicit comment POST.</p></li>
     <li><code>scaffold</code><p>New templates and promote import → responses.</p></li>
     <li><code>tui</code><p>Interactive ratatui corpus browser.</p></li>
   </ul>

Building the site
-----------------

.. code:: shell

   # needs emacs (org export), Python venv deps, and sphinx-rustdocgen (cargo install)
   ./docs/build.sh
   # open docs/build/index.html — Rust API is under docs/build/api.html and crates/

# sphinx-rustdoc-postprocess injects the crate toctree here at build time.
