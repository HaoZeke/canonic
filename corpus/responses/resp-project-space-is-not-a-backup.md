# Project space is not a backup

Hello,

Project space on the cluster is for active project data. It is more persistent than
shared scratch, but it is **not** a backup or archival store for research data
you must retain long term.

- Keep only working data on project space.
- Copy durable data to the documented archive or backup tier (for example tape
  archive) when the project needs long-term retention.
- Remove obsolete datasets so the shared quota stays available for the project.

If you need help choosing a storage class, open a ticket with the project name
and the amount of data involved. See the project-space storage SOP (URL in the
response front matter) for the formal policy checkpoints.
